# Change Note 作成スキル

実装完了後に、実装ログとして Change Note を `docs/changes/` に作成する。

## まず判断する: Change Note が必要か

以下に**該当しない**変更は Change Note を作成しない:
- 新機能の実装
- バグ修正（原因調査・対処に知見があるもの）
- リファクタリング（構造変更を伴うもの）
- テストの大幅な追加・変更

**スキップしてよい例**: typo 修正、コメント追加、フォーマット修正、設定ファイルの軽微な変更、依存関係のバージョンバンプなど、Touches/Verification/Pitfalls に記録する価値がない変更。

**該当なし → ここで終了。Change Note は作成しない。**

該当あり → 以下の手順で作成する。

## 手順

1. 今回の実装で変更したファイル・ディレクトリを把握する
2. 実施した確認手順（テスト実行など）を整理する
3. 詰まった点や注意事項があれば記録する

## 出力先

```
docs/changes/YYYY-MM-機能名.md
```

YYYY-MM は作成時点の年月。機能名はケバブケースで簡潔に。

## テンプレート

```markdown
# 機能名: 簡潔な説明

## Touches
- 変更したファイル/ディレクトリ一覧

## Verification
- 実施した確認手順

## Pitfalls（あれば）
- 詰まった点、回避策、注意事項
```
